11:T85f,<h2>The problem</h2>
<p>My personal site was running on Create React App — a tool officially archived by the React team in 2023. React 16, Stylus for CSS, deployed manually with the <code>gh-pages</code> CLI. It worked, but it wasn't something I'd point to as evidence of my front-end skills.</p>
<h2>The new stack</h2>
<p>| Concern | Before | After |
|---------|--------|-------|
| Framework | Create React App 3 | Next.js 15 |
| Language | JavaScript | TypeScript |
| CSS | Stylus | Tailwind CSS |
| Icons | FontAwesome 5 | FontAwesome 6 |
| Deploy | gh-pages CLI | GitHub Actions |</p>
<h2>Architecture</h2>
<p>The site is fully static. Next.js generates HTML at build time using <code>output: 'export'</code> and publishes the <code>out/</code> directory to the <code>gh-pages</code> branch on every push via GitHub Actions running on a self-hosted server.</p>
<pre><code>Browser → GitHub Pages (CDN)
               ↑
         GitHub Actions
        (self-hosted runner)
               ↑
         git push master
</code></pre>
<h2>The blog</h2>
<p>Posts are Markdown files in <code>src/content/posts/</code>. Frontmatter holds title, date, and description. A utility in <code>src/lib/posts.ts</code> reads them at build time using Node's <code>fs</code> module — no CMS, no API, no database.</p>
<p>To write a new post: create a <code>.md</code> file, commit, push. GitHub Actions rebuilds and deploys automatically.</p>
<h2>Scroll navigation</h2>
<p>The home page loads all posts inline. As you scroll, <code>IntersectionObserver</code> detects which post is in view and updates the URL via <code>history.pushState</code> — giving each post a shareable URL without a full page navigation. Entering the site at <code>/blog/slug/</code> renders just that post with a link back to home.</p>
<h2>What stayed the same</h2>
<p>The visual identity is unchanged: dark purple background, pink/rose text, gradient accents, dot-grid pattern. The Button component kept its animated hover effect. Same fonts (Poppins + Roboto). Only the CSS authoring method changed — from Stylus to Tailwind with CSS Modules for complex animations.</p>
