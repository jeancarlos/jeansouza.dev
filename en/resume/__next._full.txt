I needed somewhere to keep my ideas, show what I can do, and have something concrete that backs up what's on my resume. A portfolio that isn't just a list of technologies, but actually proves that what's written there is real. Because today anyone can generate code with AI. The difference is in judgment, taste, the sense of what makes sense and what doesn't. And that's not something you can fake: only someone who genuinely understands can build something that shows it. This blog is the most honest way I found to bridge that gap.

And this isn't the first one. I like the old-school aesthetic, but not how it was built. At some point I threw together some HTML and CSS on gh-pages, pushed it, and never touched the content again. Then I came back and structured a blog with CI/CD and Next.js SSG, but didn't finish that either. Never refined it, never wrote more posts. This time the technical difficulty is the point, not an accident: an interface that's genuinely unusual on the web, well implemented. That's where I'm hoping to build something other developers can read and learn from, because each post is a study of a real problem, and the code is open on GitHub for anyone who wants to audit the decisions, not just read about them.

But thinking through every detail upfront is impossible. You only find out what needs adjusting by actually using it, and the interface still has to work in the real world and meet my expectations for how users are going to navigate it. That's where decisions come from about what to skip, keep, or improve. Not everything I planned will get built: if it doesn't fit the vision, doesn't add much, or adds unnecessary complexity, it's out. Other things come in over time. One of them is a projects list, with a writeup for each one I find worth talking about — probably alongside the blog rather than a separate projects page.
